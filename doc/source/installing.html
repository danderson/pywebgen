Interested in test driving pywebgen? Cool! But before you can play with it, you'll need to install it.

Before installing

Pywebgen depends on two Python packages that are not part of Python's standard distribution: Jinja2 and PyYAML. You should install these yourself before proceeding. Most linux distros ship these packages. For example, on a Debian testing or unstable system, you can install them with: 

$ sudo apt-get install python-jinja2 python-yaml

Installing pywebgen

You can download pywebgen in one of the following formats:

  • pywebgen-0.1.0.tar.bz2
  • pywebgen-0.1.0.zip

Pywebgen is also available as a Git repository, hosted on Github. If you prefer to get the bleeding edge version, you can grab it by running:

$ git clone git://github.com/danderson/pywebgen.git

This will download a copy of pywebgen into the pywebgen directory on your machine.

If you grabbed a release package, decompress it, and set it up as you would any python package:

$ cd pywebgen-0.1.0
$ sudo python setup.py install

Once it's installed, you should be able to run pywebgen and get the help output:

$ pywebgen --help
Usage: pywebgen  [options] 

Commands:
  startsite
  generate
  vgenerate
  vcurrent
  vinfo
  vgc
  deploy
  undeploy

Options:
  --version   show program's version number and exit
  -h, --help  show this help message and exit

Running pywebgen locally

You can also run pywebgen locally without installing it, if you prefer not to. To do this, you simply need to be in the pywebgen directory when running commands, so that Python can locate the pyweb module correctly. As we'll see in the next section, this isn't actually much of a problem.
