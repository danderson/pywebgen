Pywebgen is a toolkit that helps you generate and manage a static website.

A static website is composed only of plain HTML pages and data, with no dynamic parts. It's a site that the web server can serve without thinking about it.

The problem with static websites is that there is often a lot of boilerplate: page headers, footers, menu bars... The usual solution to this involves copying and pasting the boilerplate between files, or switching to dynamic pages just for trivial "include this file" functionality.

Pywebgen solves this problem by letting you create your website as a set of template files, and then generating the actual site from these templates. This lets you write your boilerplate only once, like with dynamic languages, but still produces a completely static website that is very friendly to serve.

While I believe that pywebgen is quite easy to use, please note that it was developped with the "power user" in mind. It's not a Frontpage Express, and given its young age, it may require a little bit of tweaking to get working properly.

Features

  • Generation of HTML pages from Jinja2 templates.
  • Generation of CSS stylesheets from a small custom YAML-like language
  • Simple versionning support: a new version of the site that you generate doesn't become the "live" version until you decide to make it so.
  • Deployment support: you can deploy your static website into a directory containing other data, and later undeploy it without touching the other files in the directory.
